Data binding is one of the most significant and effective elements for creating communication
 between the DOM and the component. It makes designing interactive apps easier by reducing the
 need to worry about data pushing and pulling between the component and the template.
 There are Four types of Data binding in Angular:

Event Binding
String Interpolation
Two way data binding

2. What is Two-Way Data Binding in Angular?
Two-way data binding means that data flows in both directions:

From the component to the template (view).

From the template (user input) back to the component.

This is commonly used in form fields where you want to both display and update a value in real-time.
How It Works
Angular uses the [(ngModel)] directive to enable two-way binding. Under the hood, itâ€™s a shorthand for combining:

Property binding: [value]="..." (data â†’ view)

Event binding: (input)="..." (view â†’ data)
<input [(ngModel)]="username">
<input [value]="username" (input)="username = $event.target.value">

3.Annotations in Angular are metadata markers attached to classes using decorators 
(like @Component, @NgModule) to tell Angular how to process the class
@Component({
  selector: 'app-hello',
  template: '<h1>Hello</h1>'
})
export class HelloComponent {}

Here, @Component is an annotation that tells Angular this class is a component and provides its metadata.

In short: Annotations configure and decorate Angular classes so the framework knows what to do with them.

Dependency Injection is a design pattern where Angular provides required services or objects to a class, instead of the class creating them.

Why Use DI?
Promotes loose coupling

Improves testability and code reusability
ðŸ”§ Example
1. Create a Service
ts

@Injectable({ providedIn: 'root' })
export class LogService {
  log(msg: string) {
    console.log('LOG:', msg);
  }
}
2. Inject into a Component
ts
Copy
Edit
@Component({...})
export class AppComponent {
  constructor(private logService: LogService) {}

  ngOnInit() {
    this.logService.log('App Loaded');
  }
}
In short:
Angular DI automatically injects services into components or other services when needed,
 using constructors.
